Date: October 1, 2024

One recent project I worked on was creating a Python script that scans my disk drives and generates CSV files with details of the contents. I then built another script to upload those CSVs into SQL Server, making file management and data tracking more efficient.

The Challenge: Tracking Disk Drive Contents

Managing file inventories across multiple disk drives can be overwhelming, especially when it comes to keeping track of whatâ€™s stored where. I needed a solution that would automate this process, allowing me to quickly generate an up-to-date file inventory without manually checking through all the directories.

Developing the Drive Scan and CSV Creation Script

I turned to Python to automate this task. Using the os and csv libraries, I wrote a script that scans specified drives, gathers file information, and saves the details into a CSV file. This gave me an easily accessible record of all the files on a given drive.


import os
import csv

def scan_drive_and_create_csv(directory, output_csv):
    with open(output_csv, mode='w', newline='') as file:
        writer = csv.writer(file)
        writer.writerow(["File Path", "File Name"])

        for root, dirs, files in os.walk(directory):
            for file_name in files:
                writer.writerow([os.path.join(root, file_name), file_name])

    print(f"CSV created: {output_csv}")

# Example usage
directory_to_scan = '/path/to/your/directory'
output_csv_file = 'drive_contents.csv'
scan_drive_and_create_csv(directory_to_scan, output_csv_file)

Uploading the CSVs to SQL Server

Once the CSV files are generated, the next step is to upload them to SQL Server. This enables centralized storage and easy access to the data. Using the pandas and pyodbc libraries, I created a script to automate the CSV-to-database upload process.


import pandas as pd
import pyodbc

def upload_csv_to_sql_server(csv_file, connection_string):
    df = pd.read_csv(csv_file)
    with pyodbc.connect(connection_string) as conn:
        cursor = conn.cursor()
        for index, row in df.iterrows():
            cursor.execute(
                "INSERT INTO your_table_name (file_path, file_name) VALUES (?, ?)",
                row['File Path'], row['File Name']
            )
        conn.commit()
    print(f"{csv_file} uploaded successfully!")

# Example usage
connection_string = "Driver={SQL Server};Server=your_server;Database=your_database;UID=your_username;PWD=your_password;"
upload_csv_to_sql_server(output_csv_file, connection_string)

I had a collection of images, and while flipping through them manually was fine, I wanted to automate the process and add some randomness. The goal was to view these images as a slideshow but with a twist: seeing them in a different order every time.

Step 2: Writing the Script to Rename Files

I knew that randomizing the image order was going to require renaming the files in the folder. So, I wrote a Python script to shuffle the image files and rename them in a new sequence every time the script ran. This way, the slideshow wouldnâ€™t just be in the same order, but it would surprise me each time. Here's the script I used:

import os
import random

def rename_photos(directory):
    files = os.listdir(directory)
    random.shuffle(files)
    
    for i, file in enumerate(files):
        ext = os.path.splitext(file)[1]  # Get the file extension
        new_name = f"photo_{i + 1}{ext}"
        os.rename(os.path.join(directory, file), os.path.join(directory, new_name))

# Specify the folder path
folder_path = 'rename_photos'
rename_photos(folder_path)

This script was a game changer! It shuffled the file names and renamed them in the format photo_1.jpg, photo_2.jpg, and so on. The randomness came from using Pythonâ€™s random.shuffle() function, which means the photos now show up in different orders each time I run the script.

Step 3: Running the Slideshow

After the photos were renamed, it was time to enjoy the slideshow. Running it was easy and added a fresh layer of excitement each time I went through my images:
